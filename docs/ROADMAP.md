## 技术债

| 项 | 优先级 | 说明 |
|----|--------|------|
| 无测试 | 🔴 高 | 至少需要 service 层的单元测试 |
| OAuth 占位符 | 🔴 高 | 需要真实 client_id |
| DOM 选择器脆弱 | 🟡 中 | 需要 fallback 策略 + 定期验证 |
| 无错误追踪 | 🟡 中 | 线上问题无法感知 |
| 无 CI/CD | 🟡 中 | 手动构建+打包 |
| 无版本发布流程 | 🟢 低 | 需要 changelog + 自动发版 |
