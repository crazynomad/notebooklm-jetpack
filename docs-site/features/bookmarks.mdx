description: '多篇文章聚合为一个来源，突破 50 来源限制'
---

## 问题

NotebookLM 免费版每个笔记本只有 **50 个来源配额**。导入 20 篇博客文章就用掉将近一半。

## 解决方案

将多篇文章收藏后聚合为一个 PDF，只占 **1 个来源配额**。

<Steps>
  <Step title="收藏文章">
    浏览网页时，点击 Jetpack 图标 → 点击「收藏」按钮，将当前页面加入收藏夹。
  </Step>
  <Step title="管理收藏">
    在收藏夹 tab 中查看所有已收藏的文章。可以创建不同的**集合**（Collection），按项目或主题分类管理。
  </Step>
  <Step title="全选并导出">
    勾选要聚合的文章 → 点击「聚合导出 PDF」按钮。扩展会自动：
    1. 抓取每篇文章的正文内容
    2. 转为 Markdown 格式
    3. 合并为一份排版精美的 PDF
  </Step>
  <Step title="上传到 NotebookLM">
    将导出的 PDF 文件拖入 NotebookLM 笔记本 → **20 篇文章只占 1 个来源**。
  </Step>
</Steps>

## 集合管理

| 操作 | 说明 |
|---|---|
| 创建集合 | 点击集合名旁的 ➕ 按钮 |
| 移动书签 | 每个书签右侧的「移至...」下拉菜单 |
| 删除集合 | 删除后书签自动回到默认收藏 |
| 批量删除 | 勾选多个书签 → 点击删除按钮 |

<Tip>
  **最佳实践**：按研究主题创建集合（如「AI 论文」「产品设计」），积累到 10-20 篇时聚合导出，效率最高。
</Tip>

<Note>
  也可以直接点击「导入 NotebookLM」逐个导入，不走 PDF 聚合。适合来源配额充足的场景。
</Note>
